# Research And Data Operations

## Required Data Contract

Every displayed node score, edge pressure, micro score, trend snapshot, and future snapshot needs:

- `source_id`
- `source_status`
- `as_of`
- `collected_at`
- `applied_at`
- `cadence`
- `freshness_status`
- `observed_or_scenario`
- `calculation_status`

## Thin Path For Next Repair Cycle

`HBM -> advanced packaging -> cloud capacity`

- one entity
- one relationship
- one metric
- one official or explicitly curated source
- one update event
- one freshness rule
- one scenario rule

## Blocking Rule

Unknown, example, or missing provenance blocks representative-product promotion.
