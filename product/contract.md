## Primary Promise (정체성)

**한국인 영어 학습자가 영어의 게르만 토박이 층위 감각 — 핵심 동사(have/get/be…), 불변화사·전치사(up/out/off…), 구동사, 어순이 묘사하는 고유한 감각 — 을 문장과 퀴즈 트레이닝으로 체화하게 한다.**

- **WHO**: 한국인 영어 학습자. 첫 사용자 = 재영 본인.
- **WHY**: 한국어에는 이 감각이 생소해 영어가 번역식 처리에 머문다. 규칙 암기가 아니라 감각 체화가 목표.
- **용어 잠금**: "게르만어적 감각" = 영어 자체의 앵글로색슨 토박이 층위의 감각. 독일어가 아니다. 라틴/로망스 차용 층(receive, obtain…)과 대비되는, 일상 영어의 뼈대를 이루는 층위.

## 콘텐츠 데이터 계약 원칙

LLM이 문장·퀴즈를 생성하되, **감각 설명은 신뢰 출처(언어학·사전)에 묶인다.** 출처 없는 감각 설명은 콘텐츠로 승격 불가 — 잘못된 감각을 체화시키는 것은 마이너스 제품이다. 구체 스키마는 c1-1에서 데이터 계약 자산으로 생성.

## 샘플 Artifact (잠금 형식의 실물 예시 — 약속이 떨어지는 화면 단위)

한 문항의 실제 모양. 형식이 잠금 대상이며, 출처는 approve-4(출처 목록) 확정 전 잠정.

- **감각 설명 (출처 묶임)**: `have` — 소유·구매가 아니라 **"주어의 영역 안에 위치함"** 의 정적 감각. [출처: 잠정 — 인지언어학 소유 구문 연구 계열(예: Heine 1997 *Possession*), approve-4에서 확정]
- **훈련 문항** (감각 선택형 — 빈칸 암기형이 아님):
  > *"She **has** the window seat."* — 이 has가 그리는 그림은?
  > (a) 그녀가 좌석을 구매해 소유했다 (b) **좌석이 그녀의 영역 안에 놓여 있다** ✓ (c) 그녀가 좌석을 향해 가고 있다
- **전이 문항** (훈련에 쓰지 않은 새 문장, 같은 감각):
  > *"We **have** ten minutes before the train leaves."* — (b) 10분이 우리 영역 안에 놓여 있다 ✓

차별점 명시: 문항 형식은 **감각 선택형**(어떤 그림이 그려지는가)이며, 기존 학습 앱의 빈칸·뜻 암기형이 아니다.

1. **퀴즈 정답률 추이** — 같은 감각 축 성적이 세션을 거치며 오르는가

## Spiral 목표 (도달 기준) — **사용자 승인 (2026-06-11, approve-1)**

| # | 기준 | 판정 |
|---|---|---|
| 1 | 약속 가시성 — 첫 화면에서 무슨 감각을 훈련하는지 보이고 즉시 훈련 가능 | Intent Guardian 리뷰 |
| 2 | 체화 증거의 표면화 — 정답률 추이·전이 결과가 사용자에게 보이는 화면으로 존재 | 기계: 표면 존재 + 데이터 배선 검사 |
| 3 | 1축의 깊이 — 최소 1축 완결: 항목(예: 핵심 동사 5개 이상)마다 출처 묶인 감각 설명 + 훈련 문장 풀 + 독립 전이 세트 | 기계: 스키마·카운트·분리 검사 |
| 4 | 콘텐츠 신뢰 — 출처 없는 감각 설명 0건 | 기계: 데이터 계약 검사 |

`local_ready`는 위 6기준 + 수요 검증 패키지(target user·실제 task·최소 artifact·반증 질문·금지 claim·검증 위치 — 전부 구체값) 완성. 타인 검증은 local 루프 밖, handoff 이후.

### 겹 2 — 자산 체계 기준

매 사이클의 자산 갱신은 **"다음 후보를 더 효과적으로 — 약속 전달에 더 가깝게 — 만들 수 있는 상태"** 로 평가한다. 구체 질문: *이 자산 갱신이 다음 후보의 약속 전달력을 올리는가?* 올리지 않는 갱신(형식 정비, 기록용 문서)은 자산 기여로 인정하지 않는다. 속도 향상은 부산물.

### 상한 (Cycle 1 전 기록 — 원리 1 기계 검증 대상)

- `n_of_N`: Discovery Session(C1) 후보 상한 **N=3** (재조정 — approve-1 승인)
- `cap_reached_disposition`: 3개 후보 안에 Discovery 졸업선(아래)을 통과하는 축이 없으면 — 외부 검증 부족 / 데이터·사람 손 부족 / pivot_required 중 분류 후 종료. 무한 next_action 금지.
- **Discovery 졸업선**: 핵심축 후보들이 mechanical verdict 통과 상태로 실재하고, 최소 1축에서 감각 훈련 가설이 사용자 표면에서 검증(또는 명시 반증). 승격 기준 3(동사 5개 풀 깊이)은 Discovery가 아니라 **대표 후보 승격**의 기준 — Discovery 후보는 1항목 vertical slice로 충분. **충족 (2026-06-11, 후보 2개).**

### C2 Convergence 졸업선 — **졸업 완료 (2026-06-13, 사용자 확인)**

기준 1·2·3·5·6 충족, 기준 4는 시뮬레이션+페르소나 대체(사용자 결정, 약한 verdict — 실사용 수요는 출시 후). C3 Readiness 진입.

함정 명시: Convergence의 싸구려 버전은 *축의 나열*(have 탭 + up 탭이 한 앱에 있음)이다. 나열은 졸업이 아니다. 아래 **6기준 전부** 충족 시에만 졸업 — 일부 충족으로 "거의 됐다" 판정 금지.

| # | 기준 | 판정 방식 |
|---|---|---|
| 1 | **결합의 실재** — 축이 만나는 지점이 콘텐츠로 존재: 구동사(축③)는 핵심 동사 감각 × 불변화사 감각의 합성이다. cross-axis 문항(예: take up, get up — 해설이 두 감각의 합성으로 환원) ≥ 10개가 코퍼스·후보에 실재. 축별 탭 분리(나열)만 있으면 미충족 | 기계: cross-axis 항목 카운트·참조 검사 + 적대적 검수(합성 해설의 출처 정합) |
| 2 | **코퍼스 두께** — 핵심 동사 ≥ 5 + 불변화사 ≥ 2, 전 항목 출처 묶임 + 독립 전이 세트. 총 문항 ≥ 100으로 일일 세션(~15문항) 7일을 **새 문장으로** 지탱 (R6 — 재방문은 콘텐츠 공급) | 기계: 카운트·분리·중복 검사 |
| 3 | **유형 다양성의 실증** — 세 유형(sense-choice·verb-choice·sense-cloze)이 모두 실전 출제되고, verb-choice의 오답 동사가 감각 대비로 작동함이 검수로 확인 | 기계: 유형 분포 검사 + 적대적 검수 |
| 4 | **성장 증거의 실재** — 본인 실사용 ≥ 7세션(서로 다른 날), 전이 정답률 추이가 화면에 누적. 설계의 존재가 아니라 **기록의 존재** (약한 verdict 명시 유지 — 저자 오염 빈틈 7) | 기록: localStorage 세션 로그 export + cycle record |
| 5 | **출처 신뢰 상승** — 코어 sense(최소 동사 5개분)의 출처 사람 리뷰 완료 → `validation.strength: strong`. weak 코퍼스로는 졸업 불가 (approve-4 처리가 졸업의 전제) | 사람 리뷰 + 기계: strength 필드 검사 |
| 6 | **타인 검증 준비 완료** — 본인 외 학습자에게 전달 가능한 실행 패키지 + blind task 안내문 존재 (실행 자체는 Readiness) | 리뷰: 패키지 실재 확인 |

- 4번은 달력 시간(≥7일)이 필요한 기준이다 — 에이전트가 압축할 수 없고, 그게 의도다. 실사용 없는 Convergence 졸업은 자기기만.
- **C2 상한**: `n_of_N` **N=5** (졸업선 상향에 따라 4→5). 5개 후보에도 미충족이면 cap_reached_disposition 분류 후 정지, 사용자 판단으로.
