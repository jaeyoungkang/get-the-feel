# 훈련 설계 규칙 (정본)

후보가 훈련·퀴즈를 설계할 때 따르는 규칙. c1-1 평가(페르소나 blind task 2 + 적대적 검수)에서 회수.

## R1 — 정답 위치는 반드시 셔플 (c1-1 반증 회수)

데이터의 `answer_index`와 무관하게 **앱은 보기 표시 순서를 문항마다 셔플**해야 한다. c1-1은 전 문항 정답이 (b) 위치에 고정되어 3문항 만에 패턴으로 풀렸다(회의 페르소나 100/100 — "문장을 안 읽어도 맞힌다"). 정답 위치 패턴은 훈련을 한국어 독해 게임으로 바꾼다. verdict에 셔플 존재 검사 추가 (c1-2부터).

## R2 — 오답 보기 설계: 근접 오개념 1개 의무

보기 3개 중 오답 2개가 모두 "대놓고 틀린" 미끼(구매했다/걸어간다)면 영어 감각 없이 한국어 보기만으로 풀린다. **오답 1개는 인접 sense 또는 그럴듯한 한국인 오개념**이어야 한다 (예: 정적 위치 문항에 "사건으로 겪는다"를 오답으로). c1-1에서 가장 학습 효과가 컸던 문항(t8 problem)이 정확히 이 구조였다.

## R3 — 경계 문항은 의도적 설계 요소

직관에 반하는 항목(problem = 사건처럼 느껴지지만 정적)은 함정이 아니라 **최고 학습 지점**이다 (페르소나: "내가 틀려서 더 기억에 남는다"). 감각 축마다 경계 문항 1개 이상 의도 배치, 해설에서 반드시 `boundary_ko`(진행형 가부 등 기계적 표지)로 해소.

## R4 — 전이 세트: 표면 변형 금지

전이 문항은 훈련 문항의 어휘 치환(quick chat → long talk)이 아니라 **새 인지 맥락**(추상 주어, 시간, 성질)이어야 한다. 문장 중복 검사(기계)는 표면 변형을 못 잡는다 — 설계 시 패턴 거리를 확인.

## R5 — 감각 설명의 출처-그림 일치

sense의 `image`는 `source_refs.claim`이 실제로 보증하는 범위 안이어야 한다. 정적 출처(위치 도식)에 동적 그림("통과")을 얹는 것이 "출처 붙은 오개념"의 전형 (c1-1 검수 B1).

## R6 — 콘텐츠 공급이 재방문의 본체

게이미피케이션 없이 다시 열 이유 = 새 콘텐츠. 15문항(1항목)은 정확히 1일치다 — 두 페르소나 모두 "내일 열면 같은 문제 셔플"을 이탈 사유로 지목. 재방문 설계는 동기 장치가 아니라 **콘텐츠 공급 설계**(새 문장 추가, 다음 항목 진행)다.

## R7 — 산출 연습 부재 (인지된 한계, V3 후보)

선택형은 수용 훈련이다. 두 페르소나 모두 "고를 줄 알게 됐지만 쓸 줄은 모른다"를 지적 — 산출(쓰기·말하기) 훈련은 미해결 가치. 제품 계약 V3 후보로 등재, Discovery 범위 밖.

## R8 — 첫 세션의 진척 화면

추이 차트는 세션 1회차엔 막대 1개라 빈 화면이다. 첫 세션은 추이 대신 "오늘 잡은 감각 요약"을 보여주고, 추이는 2회차부터.
