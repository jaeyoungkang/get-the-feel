# UX 문법 — 감각 전달 디자인 규칙 (정본)

"감각"을 화면 경험으로 전달하는 규칙. 이 제품의 시장 차별점이 사는 자리.

## G1 — 공간 메타포 어휘

- **정적 위치 감각**: 주어 = 타원(영역), 대상 = 칩이 원 안에 **내려앉아 정착**하는 애니메이션. 캡션 "정적으로 자리한다".
- **사건 감각**: 주어의 하루 = 가로 띠, 사건 = 띠의 **한 칸을 차지**하는 블록. 캡션 "겪는다".
- 동사·축이 늘면 메타포 어휘도 이 문서에 추가.

## G2 — SVG 라벨은 파싱 금지, 콘텐츠 명시 필드로

문장에서 정규식으로 주어/대상을 추출하면 부사·도치에서 깨진다. 문항 데이터에 `subject_label`/`object_label` 명시 필드를 두고 그대로 표시한다.

## G3 — 메타포 반복 피로

같은 그림이 반복 누적되면 사용자는 스킵한다 (페르소나: "도움 → 장식" — 구체 횟수는 N=1 관찰이라 잠그지 않음). 대응: 문장 라벨로 변주 + 정답 직후 자동 진행 옵션 + 새 감각 진입 시에만 풀 애니메이션.

## G4 — 오답 시 시각 긴장 유지

오답에도 정착 완료 그림을 보여주면 "틀림"의 교정 신호가 죽는다 (Intent Guardian). 오답 시 미정착(칩이 원 밖에 떠 있는) 상태를 1박자 보여준 뒤 정착.

**우선순위: G4 > G3.** 반복 피로로 애니메이션을 생략하더라도 **오답이면 항상 미정착 1박자가 우선**한다 — 교정 신호는 피로 대응으로 생략할 수 없다.

## G5 — 사용자 표면 어휘: 방법론·학술 용어 금지

"체화", "전이 테스트", "근사 증거", "잠정(weak)" 같은 표현은 사용자에게 학원 광고·개발자 메모·불안 요소다. 규칙:
- 사용자 화면: "그림으로 익히기", "새 문장 테스트", "감각 점수" 같은 생활 어휘.
- 상태·검증 정보(콘텐츠 버전, 출처 상태)는 첫 화면에서 치우고 정보(ⓘ) 뒤로 — 정직성은 유지하되 전면 노출 금지.
- 정직 표기("선택형은 말하기를 증명하지 않는다")는 유지하되 사용자 언어로 ("고르기를 잘해도 말하기는 따로 늘어요 — 이 점수는 '알아보는 힘'이에요").

## G7 — get 도달 메타포

- **동적 도달 감각**: 주어의 원(영역) **밖**에서 화살표를 따라 칩이 **안으로 들어와 닿는** 그림. have(원 안 정적)와 나란히 대비시키면 "이미 안 vs 들어오는 순간"이 한 화면에서 갈린다 — verb-choice 피드백의 본진 (페르소나: "암기로 안 풀리고 감각으로 골라야").
- 상태 변화 계열은 주어가 상태(위치)로 옮겨 가 닿는 방향 — STATES ARE LOCATIONS 방향 준수 (역방향 금지).

## G8 — 합성 시각화 (cross-axis)

- 구동사 피드백: **[동사 미니 그림] + [불변화사 미니 그림] = [합쳐진 그림]** 3칸 배치 + "이미 아는 두 그림의 결합, 통암기 아님" 캡션. boundary_ko(down 반전 테스트)를 함께 — 페르소나가 "통암기엔 없던 도구"로 확인.
- **합성 캡션의 동사도 명시 필드(`verb_label`)만** — 문장 파싱 추출은 부사·명사구에서 깨진다.
- **합성 과신 경계**: 투명 합성 사례만 보여주면 사용자는 "모든 구동사가 합으로 풀린다"고 믿게 된다 (페르소나: "give up에서 깨질 것"). 구석의 자백(boundary_ko)으로 부족 — 반례(관용 구동사)의 정면 노출은 training-design R10.

## G9 — 항목별 메타포 어휘 확장

- **take**: 주어 원 안에서 손(갈고리 화살표)이 밖으로 뻗어 대상을 쥐고 끌어오는 그림 — get(대상이 저절로 들어옴)과 손의 방향으로 구분.
- **make**: 원 안에서 새 형체가 점으로 생겨나 커지는 그림 (없던 것의 생성).
- **out-exit**: 용기 안의 칩이 경계를 넘어 밖으로 / **out-reveal**: 가려진(덮개) 칩이 덮개 밖으로 나와 보이게 됨 — in 반전은 exit만 성립.
- **자물쇠 (opaque-idiom)**: 관용 구동사는 합성 3칸(G8) 금지 — 자물쇠 그림 + "통째로 익히는 게 정직" 캡션. 합성 시각화를 관용에 쓰면 R10 위반.
- **keep**: 원 안의 칩이 경계로 흘러나가려는 점선 화살표(tendency)를 손/걸쇠가 붙드는 그림 — have(칩이 그냥 놓임, 누르는 손 없음)와 나란히 두면 force 대비가 한 화면에 보인다. Day 합산 추이는 콘텐츠 풀이 달라 근사 — 화면에 근사 라벨 필수.

## G14 — 피드백 정보 위계

정답 후 피드백이 1700px+로 길어 "다음 문항"이 화면 2.3개 아래 → 3페르소나 전원 "읽을 게 너무 많다 / 그냥 다음 누르고 싶다". 규칙:
- **그림(차별점)은 항상 보이고, 진행 버튼은 그림 바로 아래.** 긴 말 설명(why_ko)·문법 메모(boundary)는 `<details>`로 접는다 — 그림에서 감을 잡은 사람은 안 펼치고 넘어간다 (그림 우선, 말은 요청 시).
- 답한 직후 피드백 head를 화면 상단으로 `scrollIntoView` — 질문 카드의 잠긴 보기를 다시 스크롤로 지나치지 않게.
- sense-choice는 질문 문장을 피드백에서 복제하지 않는다 (위 카드에 이미 있음). 빈칸 유형만 완성 문장 표시.
- 모바일 viz max-height 조임. 결과: 흔한 유형 버튼이 그림 아래 315~474px(한 화면 내), 최다 시각요소(동사 대비·합성)만 짧은 1회 스크롤.

## G13 — SVG 라벨은 폭에 맞춘다

명시 라벨 필드(G2)도 **길이는 보장 안 된다** — "held inside the yard" 같은 긴 object_label이 고정 폭 칩을 뚫고 타원과 겹쳐 모바일 레이아웃이 깨졌다 (외부 배포 후 사용자 적발). 규칙: SVG 텍스트는 항상 폭 기반 fit(글자 크기 축소 → 말줄임) 헬퍼를 거친다. 고정 font-size로 라벨을 직접 그리는 것 금지. (G2가 "파싱 금지·명시 필드"를, G13이 "명시 필드의 표시 안전"을 맡는다.)

## G10 — 색·라벨 헬퍼는 데이터와 함께

화면 헬퍼(항목 색·짧은 라벨·유형 라벨·감각 라벨)는 호출부보다 먼저 정의 존재를 확인하라. 같은 클래스 재발 시 실행 스모크나 정적 검사를 추가한다.

## G11 — 질문 단계의 정답 단서 차단

질문 카드에는 **답이 될 수 있는 메타 정보를 노출하지 않는다** — 항목명 알약(verb-choice에서 "get", sense-cloze에서 "up"이 곧 정답), 감각명, 색 단서. 이런 맥락 정보는 **정답 후 피드백으로 이동**한다.

## G12 — 한국어 해석은 정답 후에만

영어 문장의 한국어 해석(`sentence_ko`)은 **정답 후 피드백에 표시** — 정답 전 해석은 verb-choice의 답을 누설하고("받았다" → got), 번역 우회 읽기를 유도해 감각 훈련을 깎는다. 단 문장을 아예 못 읽으면 훈련이 막히므로: 정답 후 해석 표시가 기본, 어휘 난도는 A2-B1 유지로 사전 차단.

## G6 — 진척 화면의 정직 라벨

N=1·저자 본인 기록은 "효과 증거"로 오독될 수 있다 (Data/Sellability). 진척 화면에 "내 기록 (본인 학습용)" 수준의 라벨 한 줄.

## G15 — 학습 경로 선택권 + 통계

- **주제 선택**: 일일공급(R6)을 *기본*으로 두고, 한 항목을 골라 집중하는 모드를 *선택*으로 추가. 집중이 기본을 대체하면 R6(재방문=콘텐츠 공급) 약화 — 항상 기본 우선, 집중은 보강.
- **약점→집중 연결**: 통계의 강약점 카드에서 약한 감각을 바로 집중 세션으로 보내는 버튼. "약점 보고 → 그 감각만 연습"이 재방문 동선의 핵심(페르소나 검증).
- **통계 = 성장의 방향**: 점수가 아니라 "어떤 감각이 늘고(▲)/흔들리는지(▼)"를 세션 간 추이로. N=1·저자 오염은 "내 기록(본인 학습용)·효과 증거 아님" 라벨 유지(G6·빈틈7).
- **통계 과밀 차단**: 안 푼 감각(t<2) 줄이 화면을 메워 약점이 묻힌다 → 푼 감각만 기본 노출, 나머지는 "전체 보기" 접기.
- **통계 기록 함정**: 세션이 *사후 테스트까지* 마쳐야 기록이 쌓이는데 메인만 풀고 통계를 보면 빈 화면 → 김 샘. 빈 상태는 "테스트까지 마치면 쌓여요"로 안내 + 세션 미완료 중 통계 링크 비노출 + 요약 CTA를 "테스트로 마무리"로.
- **랜딩 행동 먼저**: 모드 선택·시작 버튼을 인트로 산문보다 위로. 설명은 <details>로 접어 첫 화면에 행동이 보이게.

## G16 — get+과거분사 메타포 + 변별

- get-into-state(get married/started/dressed)는 get 도달 화살표를 재사용하되 도착 칸 라벨을 "into a … state"로 — 형용사 변화(get tired)와 그림을 공유하므로 **수동/재귀 경계 패널**(get married ≠ be married by someone, get+pp는 진행 가능)로 변별한다.
- 라벨이 길어 잘림 → fitText(G13) 필수 + 칸 폭 확대.

## G17 — 산출(써보기) 모드 UX

- 인식 모드와 **나란히 선택**(써보기 추가, 대체 아님). 난이도 3단 진입.
- 피드백에 **감각 그림(메타포) 재노출** — 산출을 감각에 묶어 자유 작문 드리프트 차단(정체성).
- 빈칸/어순: 자동 채점 즉시 정오. 오답은 내 입력 vs 정답 색 대비 + why + 그림.
- 전문 쓰기: 약한 verdict 라벨(R14) + 3축 rubric 체크. 통계는 "①알아보는 힘 / ②꺼내 쓰는 힘" 분리 섹션.
- 한계 정직: 이건 *쓰기*지 *말하기*가 아니다(발화·시간압박 없음) — 페르소나 "입 여는 8할". 과대 표기 금지.
