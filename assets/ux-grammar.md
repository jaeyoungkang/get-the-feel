# UX 문법 — 감각 전달 디자인 규칙 (정본)

"감각"을 화면 경험으로 전달하는 규칙. 이 제품의 시장 차별점이 사는 자리. c1-1에서 회수.

## G1 — 공간 메타포 어휘 (c1-1에서 검증됨)

- **정적 위치 감각**: 주어 = 타원(영역), 대상 = 칩이 원 안에 **내려앉아 정착**하는 애니메이션. 캡션 "정적으로 자리한다".
- **사건 감각**: 주어의 하루 = 가로 띠, 사건 = 띠의 **한 칸을 차지**하는 블록. 캡션 "겪는다".
- 페르소나 검증: 처음 2-3회는 감각 전달에 실효 ("몸으로 느끼게 했다"). 동사·축이 늘면 메타포 어휘도 이 문서에 추가.

## G2 — SVG 라벨은 파싱 금지, 콘텐츠 명시 필드로

문장에서 정규식으로 주어/대상을 추출하면 부사·도치에서 깨진다 (c1-1: "We usually", "Let's"가 주어로 표시 → "기계가 대충 잘라 넣었네" — 신뢰 훼손). 문항 데이터에 `subject_label`/`object_label` 명시 필드를 두고 그대로 표시한다.

## G3 — 메타포 반복 피로

같은 그림이 5회 이상 반복되면 사용자는 스킵한다 (페르소나: "도움 → 장식"). 대응: 문장 라벨로 변주 + 정답 직후 자동 진행 옵션 + 새 감각 진입 시에만 풀 애니메이션.

## G4 — 오답 시 시각 긴장 유지

오답에도 정착 완료 그림을 보여주면 "틀림"의 교정 신호가 죽는다 (Intent Guardian). 오답 시 미정착(칩이 원 밖에 떠 있는) 상태를 1박자 보여준 뒤 정착.

## G5 — 사용자 표면 어휘: 방법론·학술 용어 금지

"체화", "전이 테스트", "근사 증거", "V2", "후보 c1-1", "잠정(weak)"은 사용자에게 학원 광고·개발자 메모·불안 요소다 (페르소나 양쪽 지적). 규칙:
- 사용자 화면: "그림으로 익히기", "새 문장 테스트", "감각 점수" 같은 생활 어휘.
- 상태·검증 정보(콘텐츠 버전, 출처 상태, 후보 ID)는 첫 화면에서 치우고 정보(ⓘ) 뒤로 — 정직성은 유지하되 전면 노출 금지.
- 정직 표기("선택형은 말하기를 증명하지 않는다")는 유지하되 사용자 언어로 ("고르기를 잘해도 말하기는 따로 늘어요 — 이 점수는 '알아보는 힘'이에요").

## G6 — 진척 화면의 정직 라벨

N=1·저자 본인 기록은 "효과 증거"로 오독될 수 있다 (Data/Sellability). 진척 화면에 "내 기록 (본인 학습용)" 수준의 라벨 한 줄.
